# SOLID модель холодильника ( Fridge ) 

Программная реализация холодильника с соблюдением ООП и принципом SOLID

# Технический стек

- Java: 21
- Система сборки: Maven

# Обзор проекта

- Fridge 
    - Класс который работает , только с полками . Является классом который как раз соединяет разные полки.
    - В конструктор могут передаваться абсолютно разные полки которые реализуют интрефейс *ShelfInterface*

- FridgerInfo
    - Это Record класс , который используется
    - Для заполнения паспортными данными . Так как холодильника не заботит , есть у него имя или модель , поэтому вынеслось в отедльный класс
    - Так же имеет метод , который возвращает паспортные 
        - Можно было это сделать и в самом классе холодильника . Но тогда будет нарушение принципа SOLID . 
        - Поэтому была созданна специальная функция которая как раз этим и занимается

- Enum data-class ( PowerStatus , DoorStatus)
    - Простое перечисление , которая задает какие будут режими работы и положения двери у холодильника
        
- ShelfFactory
    - Это реализация принципа Фабрика , которая позволяет создавать разные классы ( отличные от Shelf) , но которые реализуют одинаковый интрефейс ( ShelfInterface )  
    - В нем как раз описан принцип создания объекта Shelf

- Shelf
    - Это конкретная реализация интрефейса ShelfInterface. 
    - Но никто не запрещается создать абсолютно другую полку , которая будет не похожа на Shelf , и при этом все будет работать

- Food
    - Базовый класс с помощью которого будет создаваться еда .
    - Этими объектами будет наполняться объекты созданные на основе ShelfInterface 

- Expirable
    - Интерфейс , который обязаться работу с датами

- Exceptions
    - Кастомные ошибки

# Как запускать

У вас должен быть установлен Maven.

```bash
git clone https://github.com/ArcxExe/Project-For-SpeedOfLight.git` # Клонирование проекта
cd Project-For-SpeedOfLight/ && mvn clean package && java -jar target/solid-fridge-1.0-SNAPSHOT.jar` # Запуск
```
